\chapter{Theory of Computation}
\label{cha:theory_of_computation}


In Theory of Computation, various models of computation with different levels of expressive power are studied. To discuss and compare different levels of expressive power, it is inevitable to consider algorithms in the context of some application domain. The typically chosen application domain is the syntax of formal languages.
There are two techniques for defining languages that are typically used. The first technique is based on the concept of generation and uses the formalism of grammars. Another technique is based on the mechanism of recognition and uses the formalism of automata.


\section{Chomsky Hierarchy}
\label{sec:chomsky_hierarchy}

Noam Chomsky (with the help of Marcel-Paul Schützenberger) identified several classes of formal languages, with different levels of specification complexity, requiring computational models with different levels of expressive power to solve the problem of recognizing these languages \cite{cit9}.

There are four levels: type 0, type 1, type 2, and type 3.

\begin{itemize}
  \item Type 0 — Unrestricted Grammars
  \item Type 1 — Context-Sensitive Grammars
  \item Type 2 — Context-Free Grammars
  \item Type 3 — Regular Grammars
  
\end{itemize}

\begin{figure}[ht]
  \includegraphics[height=6cm]{Chapters/Figures/Chapter2/tese1.png}
  \centering
  \caption{Chomsky hierarchy}
  \label{fig:tese1}
\end{figure}

The Chomsky hierarchy involves four levels of language types. Each type of language is associated with a type of generator and a type of recognizer.

\subsection{Type 0: Unrestricted Grammars}

Type 0 grammars, also known as unrestricted grammars, are the most general class in the Chomsky hierarchy. They have no restrictions on their production rules, meaning that any string of symbols can be replaced by any other string of symbols. This flexibility allows Type 0 grammars to generate any language that is recognizable by a Turing machine, making them equivalent in power to Turing machines. In other words, they can describe all recursively enumerable languages \cite{cit9}.

An example of an unrestricted grammar is:


\[ G = ( \{S, A, B\}, \{a, b\}, P, S ) \]
\[ P = \{ S \rightarrow aAB, AB \rightarrow BA, A \rightarrow a, B \rightarrow b \} \]

Derivation of the word "aba":
\[
\begin{aligned}
S &\rightarrow aAB \\
aAB &\rightarrow aBA \\
aBA &\rightarrow abA \\
abA &\rightarrow aba
\end{aligned}
\]

\subsection{Type 1: Monotonic Grammars}

On type 1 grammars, or monotonic grammars, 
the length of the string on the left-hand side of a 
production rule is less than or equal to the length of the 
string on the right-hand side. This property ensures that the 
derivation process does not shorten the string. Type 1 grammars 
can generate languages 
that are recognized by a linear bounded automaton, which is a 
Turing machine with a tape of limited length 
proportional to the input size \cite{cit9}. 

An example of a monotonic grammar is:

\[ G = ( \{S, B \}, \{a, b, c\}, P, S ) \]
\[ P = \{ S \rightarrow aBSc, S \rightarrow abc, Ba \rightarrow aB, Bb \rightarrow bb \} \]


Derivation of the word "aaabbbccc":
\[
\begin{aligned}
S &\rightarrow aBSc \\
aBSc &\rightarrow aBaBScc \\
aBaBScc &\rightarrow aBaBabccc \\
aBaBabccc &\rightarrow aBaaBbccc \\
aBaaBbccc &\rightarrow aaBaBbccc \\
aaBaBbccc &\rightarrow aaaBBbccc \\
aaaBBbccc &\rightarrow aaabbbccc 
\end{aligned}
\]

\subsection{Type 2: Context-Free Grammars}
\label{subsuc:type2}

On type 2 grammars, or context-free grammars, the left-hand side
consists of a single 
non-terminal symbol, and the right-hand side can be any string 
of terminal and/or non-terminal symbols. These grammars allows 
context-free grammars to generate languages that can be 
recognized by a pushdown automaton, which is a type of automaton 
that uses a stack to manage additional information \cite{cit9}. 

An example of a context-free grammar is:

\[ G = ( \{S\}, \{a, b\}, P, S ) \]
\[ P = \{ S \rightarrow aSb, S \rightarrow ab \} \]

Derivation of the word "aabb":
\[
\begin{aligned}
S &\rightarrow aSb \\
aSb &\rightarrow aaSbb \\
aaSbb &\rightarrow aabb
\end{aligned}
\]


\subsection{Type 3: Regular Grammars}

On type 3 grammars, also known as regular grammars, represent 
the most constrained category within the Chomsky hierarchy. 
The production rules for these grammars are quite specific, 
each rule's left-hand side is a single non-terminal symbol, 
while the right-hand side is either a single terminal symbol 
or a terminal symbol followed by a single non-terminal symbol. 
This strict format allows regular grammars to generate languages
that can be recognized by finite automata. \cite{cit9}. 

An example of a regular grammar is:

\[ G = ( \{S\}, \{a, b\}, P, S ) \]
\[ P = \{ S \rightarrow aS, S \rightarrow b \} \]

Derivation of the word "aaab":
\[
\begin{aligned}
S &\rightarrow aS \\
aS &\rightarrow aaS \\
aaS &\rightarrow aaaS \\
aaaS &\rightarrow aaab
\end{aligned}
\]

This dissertation will focus on a extension of type 2 grammars: Attribute Grammars.

\section{Attribute Grammars}

Attribute grammars are an extension of context-free 
grammars that associate attributes with the grammar symbols 
and define rules for computing these attributes. 
These attributes provide a way to carry semantic information 
through the parse tree, enabling context-sensitive analysis \cite{knuth1968semantics}.

There are two types of attributes:

\begin{itemize}
  \item \textbf{Synthesized Attributes}: These are computed from the attributes of the children nodes in the parse tree. They propagate information up the tree.
  \item \textbf{Inherited Attributes}: These are computed from the attributes of the parent and/or sibling nodes. They propagate information down and across the tree.
\end{itemize}

By extending context-free grammars with attributes, attribute grammars enable more powerful and flexible language descriptions. 
In fact it is easy to integrate contextual aspects into a grammar, often more intuitively than through a monotonic grammar.
For example, instead of needing 10 rules, we can use an attribute to achieve the same result.

\subsection{Synthesized Attributes}

Synthesized attributes are computed from the attributes of the children of a node in the derivation tree. These attributes propagate information up the tree, allowing for the evaluation of properties that depend on the structure of the subtree rooted at a particular node \cite{knuth1968semantics}.

For example, consider the following grammar:

\[ G = ( \{S, A\}, \{a, b\}, P, S ) \]
\[  P = \{ S \rightarrow A, A \rightarrow aA | b \} \]
  
We can define a synthesized attribute \( \text{length} \) that calculates the length of the generated string. The rules for computing this attribute are as follows:
  
  
\[  \text{length}(S) = \text{length}(A) \]
\[  \text{length}(A \rightarrow aA) = 1 + \text{length}(A) \]
\[ \text{length}(A \rightarrow b) = 1 \]

Derivation of the word "aaab":

\[S \Rightarrow A , \quad \text{length}(S) = \text{length}(A)\] 
\[ A \Rightarrow aA , \quad \text{length}(A) = 1 + \text{length}(A)  \]
\[  aA \Rightarrow aaA , \quad \text{length}(A) = 1 + (1 + \text{length}(A))  \]
\[  aaA \Rightarrow aaaA , \quad \text{length}(A) = 1 + (1 + (1 + \text{length}(A)))  \]
\[  aaaA \Rightarrow aaab , \quad \text{length}(A) = 1  \]


Back-substituting:
\[
\begin{aligned}
\text{length}(A) &= 1 \\
\text{length}(A) &= 1 + 1 = 2 \\
\text{length}(A) &= 1 + 2 = 3 \\
\text{length}(A) &= 1 + 3 = 4
\end{aligned}
\]

Thus, the length of the generated string \( \mathbf{aaab} \) is:
\[
\text{length}(S) = 4
\]
  
In this example, the attribute \( \text{length} \) is synthesized because it is computed based on the attributes of the children nodes. For the production \( A \rightarrow aA \), the length is calculated by adding 1 to the length of \( A \). For the production \( A \rightarrow b \), the length is simply 1, as \( b \) is a terminal symbol.

\subsection{Inherited Attributes}

Inherited attributes are passed from parent nodes to child nodes in the derivation tree. These attributes propagate information down and across the tree, allowing for context-sensitive analysis \cite{knuth1968semantics}.

For example, consider the following grammar:

\[ G = ( \{S, A\}, \{a, b\}, P, S ) \]
\[ P = \{ S \rightarrow A, A \rightarrow aA | b \} \]

We can define an inherited attribute \( \text{prefix} \) that accumulates the prefix of the generated string. The rules for computing this attribute are as follows:

\[ \text{prefix}(S) = "" \]
\[ \text{prefix}(A \rightarrow aA) = \text{prefix}(A) + "a" \]
\[ \text{prefix}(A \rightarrow b) = \text{prefix}(A) + "b" \]

Derivation of the word "aaab":

\[S \Rightarrow A , \quad  \text{prefix}(S) = "" \]
\[ A \Rightarrow aA , \quad  \text{prefix}(A) = \text{prefix}(A) + "a" \]
\[  aA \Rightarrow aaA , \quad \text{prefix}(A) = \text{prefix}(A) + "a" \]
\[  aaA \Rightarrow aaaA , \quad \text{prefix}(A) = \text{prefix}(A) + "a" \]
\[  aaaA \Rightarrow aaab , \quad  \text{prefix}(A) = \text{prefix}(A) + "b" \]

Back-substituting:
\[
\begin{aligned}
\text{prefix}(S) &= "" \\
\text{prefix}(A) &= \text{prefix}(S) + "a" = "a" \\
\text{prefix}(A) &= "a" + "a" = "aa" \\
\text{prefix}(A) &= "aa" + "a" = "aaa" \\
\text{prefix}(A) &= "aaa" + "b" = "aaab"
\end{aligned}
\]

Thus, the accumulated prefix of the generated string is:
\[
\text{prefix}(S) = "aaab"
\]

In this example, the attribute \( \text{prefix} \) is inherited because it is computed based on the attributes of the parent node. For the production \( A \rightarrow aA \), the prefix is calculated by appending "a" to the prefix of \( A \). For the production \( A \rightarrow b \), the prefix is calculated by appending "b" to the prefix of \( A \).




\subsection{Circularity}

For an attribute grammar to be well-formed, the attributes associated with nonterminals at any node in the parse tree must be evaluable based 
on the grammar's semantic rules \cite{gordon}. Since non-trivial context-free grammars can generate an infinite number of parse trees, determining 
well-formedness is a complex task. The primary challenge lies in circular dependencies within the attribute dependency graph. 
For instance, if attribute a1 depends on a2, which in turn depends on a1, neither attribute can be evaluated, leading to an evaluation deadlock.

\section{Constrained Attribute Grammars}

Constrained Attribute Grammars (CAG) further enhance attribute grammars by incorporating constraints that capture spatial 
relationships between symbols \cite{giulia}. This is essential for recognizing multi-dimensional objects, such as mathematical formulas or organizational 
charts, which cannot be easily represented as one-dimensional strings.

Each production rule in a CAG can be divided in three components \cite{giulia}, syntatic part, semantic part, and constraints.
The syntatic part defines the structure of the rule, e.g.,E→E1E2...En. On other side the semantic part specifies how to compute the attributes of the 
left  side symbol based on the attributes of the right side symbols. Finnaly the constraints establish relationships between input symbols and 
rule symbols, often expressed in terms of attributes.

The parsing algorithm for CAGs leverages the concepts of keywords and relevance measures to efficiently parse multi-dimensional objects. 
The algorithm proceeds by selecting the most relevant keyword, applying constraints to partition the input symbols, and recursively parsing 
the subsets of input symbols corresponding to the non-terminal symbols in the rule \cite{giulia}.

\subsection{Example of a Constrained Attribute Grammar }

Consider a simple CAG for recognition mathematical expressions involving addition and division in figure \ref{fig:figure4}

\begin{figure}[ht]
  \includegraphics[width=\textwidth, height=\textheight, keepaspectratio]{Chapters/Figures/Chapter2/figure4.png}
  \centering
  \caption{Production rules fi~r a CAG for simple mathematical expressions \cite{giulia}}
  \label{fig:figure4}
\end{figure}

If we do the parsing of the mathematical expression: 3 + 5 / 2

\[
\begin{aligned}
S \Rightarrow E, \quad A(S) = A(E) \\
E \Rightarrow E_1 + E_2, \quad A(E) = A(+) \cup A(E_1) \cup A(E_2) \\
E_1 \Rightarrow 3, \quad A(E_1) = A(3) \\
E_2 \Rightarrow E_3 / E_4, \quad A(E_2) = A(/) \cup A(E_3) \cup A(E_4) \\
E_3 \Rightarrow 5, \quad A(E_3) = A(5) \\
\quad E_4 \Rightarrow 2, \quad A(E_4) = A(2) \\
\end{aligned}
\]

Now, let's back-substitute the attributes:

\[
\begin{aligned}
&A(E_4) = A(2) \\
&A(E_3) = A(5) \\
&A(E_2) = A(/) \cup A(5) \cup A(2) \\
&A(E_1) = A(3) \\
&A(E) = A(+) \cup A(3) \cup (A(/) \cup A(5) \cup A(2)) \\
\end{aligned}
\]

Thus, the semantic attributes of the parsed string \(3 + 5 / 2\) are:

\[
A(S) = A(E) = A(+) \cup A(3) \cup A(/) \cup A(5) \cup A(2)
\]



\section{Automata}
Automata theory is a fundamental area of theoretical computer science that deals with the study of abstract machines and the problems they 
can solve \cite{cit4}. As described in the Chomsky hierarchy, different types of automata correspond to different classes of formal languages.

\subsection{Types of Automata}

\subsubsection{Finite Automata}
Finite automata (FA) are  models in computer science used to recognize regular languages, they consist of a finite number of states, 
transitions between these states, and an input alphabet, when an input string is processed, the automaton transitions through states 
according to the input symbols. If it ends in an accepting state, the string is recognized as part of the language.

Due to their simplicity and efficiency, this models can be used in various applications such as in text processing, 
lexical analysis and in network protocols to manage and control the flow of data.

\subsubsection{Pushdown Automata}
Pushdown automata (PDA) are more powerful than finite automata and can recognize context-free languages. 
They extend finite automata by adding a stack, which provides additional memory, this stack allows PDAs to 
handle nested structures, such as parentheses in arithmetic expressions or nested loops in programming languages and 
used in parsing and syntax analysis in compilers \cite{cit5}.

\subsubsection{Turing Machines}
Turing machines (TM) are the most powerful type of automata and can recognize recursively enumerable languages, they consist of an infinite tape,
 a tape head that can read and write symbols, and a set of states. Turing machines can simulate any algorithm and are used to define the limits 
 of what can be computed and are fundamental in the study of computability and complexity theory \cite{cit5}.

\subsubsection{Linear Bounded Automata}
Linear bounded automata (LBA) are a type of Turing machine with a tape of limited length proportional to the input size, they recognize 
context-sensitive languages. LBAs are more powerful than PDAs but less powerful than general Turing machines, they also are used in certain 
types of language recognition and computational complexity studies \cite{cit5}.
